UI/UX Design Review

Design Analysis:
HanapBuhay Job Portal

Consistency, Visual Hierarchy, Layout, and Navigation Design Review

HanapBuhay — Modern Job Portal April 8, 2026

1 Consistency and Standards

1.1 Color Palette

The website maintains a consistent color system defined through CSS custom properties in world-class.css:

TokenValueUsage
--wc-bg #f8fafc Page background
--wc-surface #ffffff Card / container backgrounds
--wc-text #0f172a Primary text (headings)
--wc-muted #475569 Secondary / paragraph text
--wc-primary #2563eb Primary brand blue
--wc-primary-2 #4f46e5 Secondary brand indigo
--wc-border #e2e8f0 Default border color

These tokens are referenced across all pages, ensuring a unified Tailwind-inspired slate/indigo palette. The primary gradient linear-gradient(135deg, #3b82f6, #6366f1) is reused in the hero search button, the “Post Job” CTA button, navbar hover underlines, footer subscribe button, and job card accents — creating a strong brand identity throughout the site. There is a minor inconsistency: some older inner-page heroes (About, Contact, Privacy) use a slightly different gradient overlay approach (radial-gradient with rgba(99,102,241,.22)) while the index hero uses a distinct animated-shapes approach. However, the overall color feel remains cohesive.

1.2 Typography

All pages share a single font stack defined globally in the body rule:

font-family: Inter, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

Headings consistently use letter-spacing: -0.02em and font-weight: 800 for section titles, 700 for card titles, and 600 for nav links and labels. This three-tier weight system is applied uniformly across the index page (hero title, section titles, job card titles), the Companies page (.ch-title, .company-title), the About page (.section-title, .stat-box .number), and the Contact page (.contact-label). The readable font toggle in the accessibility widget offers Verdana as an alternative, maintaining legibility standards.

1.3 Button Styles

Buttons follow a consistent design language set in world-class.css:

  • Border-radius: 10px globally (pill 999px for specific CTAs like .btn-post, .btn-gradient)
  • Font-weight: 600 across all button variants
  • Primary buttons: Use the brand gradient background with no border
  • Outline buttons: Use #818cf8 or #2563eb border with transparent background
  • Hover effect: Consistent brightness(1.05) filter and translateY(-2px) lift

This is applied uniformly to the search button, “View All Jobs,” pagination links, “View Profile” buttons on company cards, and the Contact form submit button.

1.4 Spacing and Section Structure

The world-class.css enforces consistent section spacing with .site-section { padding-top: 4.5rem; padding-bottom: 4.5rem; }. All pages use Bootstrap’s .container with a max-width: 1200px cap. Card border-radius is standardized at 14px (--wc-radius) for .card, .job-post-item, .rj-card, .sv-card, and .emp-card. The Companies page cards use the same 14px radius. This creates a visually predictable rhythm as users navigate between pages.

1.5 Navigation Menu

The navigation bar is identical across every page because it is loaded from a single shared PHP include file (includes/header.php). It features:

  • Logo on the left
  • Menu items: Home, About, Contact, Privacy, Fresh Grads, FAQs, Companies, Explore Jobs
  • Active page highlighting via a PHP $currentPage variable and an active class
  • Consistent underline hover animation (gradient scaleX from right to left)
  • User chip dropdown (avatar + name) for logged-in users with pill border-radius
  • Notification bell with red badge counter
  • Mobile hamburger menu with slide-in drawer

The footer is equally shared via includes/footer.php, presenting four consistent columns (For Job Seekers, For Employers, Company, Contact) with a dark #0b1220 background and gradient newsletter bar. Every page loads the same footer structure.

2 Visual Hierarchy

The site demonstrates strong visual hierarchy:

Hero sections command attention with large bold headings (clamp(2.5rem, 6vw, 4.5rem) on the index, 3rem on inner pages), high-contrast white text on dark overlays, and background imagery. This immediately tells the user which page they are on and what the primary action is.

Section badges (e.g., “Fresh Opportunities,” “AI-Powered”) use small pill-shaped labels with indigo backgrounds positioned above section titles, creating a clear scanning pattern: badge → title → subtitle → content.

Job cards use a grid layout with a clear information hierarchy: company logo → job title (bold, 700) → company name and location (gray, #64748b) → posted date (small, lighter) → deadline chip (color-coded: green, amber, red). The eye naturally flows from the most important information (what job) to the least important (deadline details).

CTA buttons use the bright primary gradient to stand out from the neutral backgrounds, ensuring call-to-action elements are the most visually prominent interactive elements on any page.

Platform statistics use large bold numbers with gradient text coloring to draw attention to social proof metrics.

3 Layout Design

The layout follows a consistent structural pattern across all pages:

#LayerDetails
1Shared navbarSticky, blurred backdrop, consistent across all pages
2Hero sectionFull-width, dark overlay with centered content
3Content sectionsContainerized, 1200px max, alternating white/light gray backgrounds
4Shared footerDark four-column grid with newsletter bar

This F-pattern / Z-pattern-friendly layout ensures users always know where they are. The index page uses a card-based grid (col-md-6 col-lg-4) for companies and a list-based layout for job listings — both responsive via Bootstrap’s 12-column grid system. The site is fully responsive: cards stack to single-column on mobile, the navbar collapses to a hamburger menu, and the hero text scales down with clamp() and media queries at 992px, 768px, and 576px breakpoints.

4 Navigation Design

  • Primary navigation: Top navbar with 8 main links, always visible on desktop, collapsible on mobile. Active state is clearly indicated with the active class and a blue underline.
  • Secondary navigation: User dropdown menu provides profile, dashboard, saved/applied jobs, and logout access.
  • Utility navigation: Notification bell with real-time badge count, breadcrumbs on inner pages (About, Contact, Privacy).
  • In-page navigation: The index page hero includes a smooth-scroll indicator (“Scroll”) pointing to the jobs section. Trending search chips provide quick keyword navigation. The Companies page has a segmented filter (All / Hiring now) and a live client-side search bar.
  • Accessibility navigation: A floating accessibility widget (bottom-left) provides dark mode, font size, and readable font controls with keyboard support (tabindex, ARIA roles, Escape key to close).

Areas for Improvement

  • A few inner pages (Privacy, Companies) define page-specific styles via inline <style> blocks rather than external CSS files, which slightly reduces maintainability.
  • Some hero sections use slightly different overlay techniques (animated shapes vs. radial gradients), though the visual result is similar.
  • Inline style attributes on some elements (e.g., style="background: #f8fafc") could be replaced with utility classes for better consistency.

Conclusion

Overall, the HanapBuhay website demonstrates a high degree of consistency and standards compliance in its design. The use of CSS custom properties, shared PHP includes for navigation and footer, a unified color palette, standardized typography, and consistent card/button styling ensures that users experience a cohesive interface across all pages. The visual hierarchy is well-structured, guiding the user’s eye from hero sections to content to calls-to-action. The layout is responsive and predictable, and navigation is clear with proper active-state indicators and accessibility features. The design successfully follows established UI/UX principles that promote usability, learnability, and trust.
